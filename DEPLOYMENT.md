# 网站部署指南

## 快速部署方法

### 方法一：GitHub Pages（免费，推荐）

#### 步骤：
1. **创建GitHub账户**（如果还没有）
   - 访问 https://github.com
   - 注册新账户

2. **创建新仓库**
   - 点击右上角"+" → "New repository"
   - 仓库名：`yang-lawyer-website`
   - 选择"Public"（公开）
   - 勾选"Add a README file"

3. **上传网站文件**
   - 在仓库页面点击"Add file" → "Upload files"
   - 将`杨律师民商事诉讼网站`目录下的所有文件拖入
   - 点击"Commit changes"

4. **启用GitHub Pages**
   - 进入仓库 → Settings → Pages
   - 在"Source"部分选择"main"分支
   - 点击"Save"
   - 等待几分钟，会显示您的网站地址（如：`https://[用户名].github.io/yang-lawyer-website/`）

5. **自定义域名（可选）**
   - 购买域名（如：yanglawyer.com）
   - 在域名服务商添加CNAME记录指向GitHub Pages
   - 在GitHub Pages设置中添加自定义域名

### 方法二：Netlify（免费，更简单）

#### 步骤：
1. **访问Netlify**：https://www.netlify.com
2. **注册账户**：可以使用GitHub账户直接登录
3. **拖拽部署**：
   - 将`杨律师民商事诉讼网站`文件夹拖到Netlify的部署区域
   - 自动生成网站地址（如：`https://[随机名称].netlify.app`）
4. **自定义域名**：
   - 在Domain settings中添加自定义域名
   - 按照指引设置DNS

### 方法三：Vercel（免费，适合开发者）

#### 步骤：
1. **访问Vercel**：https://vercel.com
2. **导入Git仓库**：
   - 连接GitHub账户
   - 导入您的网站仓库
3. **自动部署**：
   - Vercel会自动检测并部署
   - 生成网站地址（如：`https://[项目名].vercel.app`）

## 常见问题

### 1. 网站无法访问
- 检查部署平台的状态
- 确认文件已正确上传
- 检查域名DNS设置

### 2. 样式不生效
- 清除浏览器缓存（Ctrl+F5）
- 检查CSS文件路径
- 查看浏览器控制台错误

### 3. 表单不工作
- 检查JavaScript控制台错误
- 确认网络连接
- 测试不同的浏览器

### 4. 移动端显示问题
- 使用响应式设计检查工具
- 测试不同尺寸的设备
- 调整CSS媒体查询
